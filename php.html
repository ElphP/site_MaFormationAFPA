Le langage PHP

Par où commencer avec PHP?

PHP est un langage de script interprété côté serveur, souvent désigné comme un langage "back-end". En d'autres termes, les scripts PHP sont compilés en éléments lisibles par un navigateur côté client (du HTML).

Voici quelques-unes des utilisations principales de PHP :

  • Générer des pages avec du contenu dynamique.
  • Créer, ouvrir, lire, écrire, effacer et fermer des fichiers sur le serveur.
  • Récupérer les informations d'un formulaire.
  • Envoyer et recevoir des cookies.
  • Effectuer des opérations CRUD (Create, Read, Update, Delete) sur une base de données : ajouter, lire, modifier et supprimer des données.
  • Contrôler l'accès des utilisateurs.
  • Chiffrer des données.

Pour commencer à développer en PHP, vous devez d'abord installer un serveur local sur votre ordinateur pour compiler le code PHP. À l'AFPA, nous avons utilisé XAMPP, qui inclut MySQL comme système de gestion de base de données relationnelles (SGBDR).

Il est important de noter que PHP est un langage non typé. Cela signifie que les variables peuvent être de différents types et qu'elles n'ont pas besoin d'être déclarées avant d'être utilisées.
Une variable est indiquée par le signe $.

Pour que le serveur puisse reconnaître le code PHP à compiler, celui-ci doit être encadré par <?php et ?> (la partie fermante peut être omise si tout le reste du code est du PHP).

Le code peut être écrit sous forme "spaghetti" c'est à dire que le script peut être "injecté" dans certaines parties d'un code HTML pour rendre cette page dynamique, mais attention tout fichier comportant du PHP doit avoir l'extension .PHP pour pouvoir être interprété!

Enfin, en PHP, la concaténation est possible avec le .

Ecercice 3.1 : include, $_SERVER["PHP_SELF"], preg_match($pattern,$var), explode("separator",$var)
POST et Cookie

Les instructions include("URLFichier"), include_once("URLFichier"), require("URLFichier") et require_once("URLFichier") permettent de rendre le PHP très modulaire. include() permet d'insérer le code d'un autre fichier mais n'empêche pas l'exécution du script si celui-ci ne peut pas être chargé, contrairement à require() qui provoquera une erreur si le chargement échoue.
_once permet d'éviter que le script appelé soit chargé plusieurs fois. (ce qui peut provoquer des erreurs)

action='$_SERVER["PHP_SELF"]' dans un formulaire HTML indique que le formulaire sera soumis au même fichier PHP qui le contient, ce qui est souvent utilisé pour le traitement du formulaire.

La fonction preg_match($pattern,$var) renvoie true si la variable $var est en concordance avec la regex $pattern (sinon elle renvoie la valeur false).

La fonction explode("separator",$var) prend deux arguments : le premier est le séparateur utilisé pour diviser la chaîne $var, et le deuxième est la chaîne à diviser. Elle renvoie un tableau contenant les sous-chaînes résultantes après avoir divisé la chaîne $var à chaque occurrence du séparateur spécifié. Dans notre cas elle récupère dans un tableau les différentes données de la date entrée sous la forme JJ/MM/YYYY: [JJ,MM,YYYY]
Enfin, la fonction "algo_verif_date" permet de vérifier si la date entrée existe... (en tenant compte des années bissextiles!)

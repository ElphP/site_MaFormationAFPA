Le langage PHP

Par où commencer avec PHP?

PHP est un langage de script interprété côté serveur, souvent désigné comme un langage "back-end". En d'autres termes, les scripts PHP sont compilés en éléments lisibles par un navigateur côté client (du HTML).

Voici quelques-unes des utilisations principales de PHP :

  • Générer des pages avec du contenu dynamique.
  • Créer, ouvrir, lire, écrire, effacer et fermer des fichiers sur le serveur.
  • Récupérer les informations d'un formulaire.
  • Envoyer et recevoir des cookies.
  • Effectuer des opérations CRUD (Create, Read, Update, Delete) sur une base de données : ajouter, lire, modifier et supprimer des données.
  • Contrôler l'accès des utilisateurs.
  • Chiffrer des données.

Pour commencer à développer en PHP, vous devez d'abord installer un serveur local sur votre ordinateur pour compiler le code PHP. À l'AFPA, nous avons utilisé XAMPP, qui inclut MySQL comme système de gestion de base de données relationnelles (SGBDR).

Il est important de noter que PHP est un langage non typé. Cela signifie que les variables peuvent être de différents types et qu'elles n'ont pas besoin d'être déclarées avant d'être utilisées.
Une variable est indiquée par le signe $.

Pour que le serveur puisse reconnaître le code PHP à compiler, celui-ci doit être encadré par <?php et ?> (la partie fermante peut être omise si tout le reste du code est du PHP).

Le code peut être écrit sous forme "spaghetti" c'est à dire que le script peut être "injecté" dans certaines parties d'un code HTML pour rendre cette page dynamique, mais attention tout fichier comportant du PHP doit avoir l'extension .PHP pour pouvoir être interprété!

Enfin, en PHP, la concaténation est possible avec le .

Les instructions include("URLFichier"), include_once("URLFichier"), require("URLFichier") et require_once("URLFichier") permettent de rendre le PHP très modulaire. include() permet d'insérer le code d'un autre fichier mais n'empêche pas l'exécution du script si celui-ci ne peut pas être chargé, contrairement à require() qui provoquera une erreur si le chargement échoue.

action='$_SERVER["PHP_SELF"]' dans un formulaire HTML indique que le formulaire sera soumis au même fichier PHP qui le contient, ce qui est souvent utilisé pour le traitement du formulaire.
Il existe une autre superglobale $_SERVER['SCRIPT_NAME'] presque similaire à $_SERVER["PHP_SELF"], mais elle retourne uniquement le nom du script, excluant les variables GET et POST le cas échéant (ce que $_SERVER['PHP_SELF'] renvoie). De plus, $_SERVER['SCRIPT_NAME'] renvoie le nom du script même s'il est inclus dans un autre, tandis que $_SERVER['PHP_SELF'] renverra toujours le chemin du script appelant.

$_SERVER['REQUEST_METHOD'] : Cette superglobale contient la méthode de requête HTTP utilisée pour accéder à la page en cours. Elle est généralement utilisée pour déterminer si une page a été demandée via une méthode POST ou GET. Les valeurs possibles incluent "GET", "POST", "PUT", "DELETE", etc.
setcookie() : Cette fonction est utilisée pour définir un cookie dans le navigateur du client. Un cookie est un petit fichier stocké sur l'ordinateur du client qui contient des données envoyées par le serveur. La fonction setcookie() prend plusieurs paramètres tels que le nom du cookie, sa valeur, sa durée de vie, le chemin, le domaine, etc.
$_COOKIE : Cette superglobale contient les cookies envoyés par le client au serveur dans la requête HTTP. Elle est utilisée pour accéder aux cookies définis précédemment par le serveur ou par le client. Les données de cookies sont accessibles via les clés de tableau associatives de $_COOKIE.
header() : Cette fonction est utilisée pour envoyer des en-têtes HTTP bruts au client. Elle est souvent utilisée pour rediriger les utilisateurs vers une autre page, définir le type de contenu, ou envoyer des en-têtes de réponse personnalisés. L'en-tête doit être envoyé avant que tout contenu ne soit envoyé au navigateur, sinon une erreur sera générée.

Un petit mot sur l'opérateur de résolution de portée :: (appelé également Paamayim Nekudotayim): Cet opérateur est utilisé pour accéder aux membres statiques, constantes et méthodes d'une classe, ainsi que pour faire référence à la classe parente dans le contexte de l'héritage (comme dans notre cas ici).
Les niveaux d'accès (public, protected, private) sont utilisés pour contrôler la visibilité des propriétés et méthodes. Par défaut, une propriété ou méthode est publique, accessible de n'importe où dans l'application (équivaut à public). En utilisant protected, l'accès est limité aux classes "filles" (définies avec extends), tandis que private restreint l'accès à l'intérieur de la classe elle-même, assurant une encapsulation complète.

new PDO($dsn, "root", "", $option) crée un nouvel objet PDO en utilisant le constructeur de la classe PDO. Explication de chaque paramètre :
$dsn variable sous forme de chaîne de caractères qui définit les détails de la connexion à la base de données. DSN signifie "Data Source Name". Ici, elle contient les informations suivantes: l'hôte, le port et le nom de la base de données.
"root" C'est le nom d'utilisateur utilisé pour se connecter à la base de données. Dans cet exemple, il est défini comme "root", qui est un nom d'utilisateur par défaut pour la base de données MySQL. (à changer lors d'une mise en production )
"" C'est le mot de passe associé à l'utilisateur. Dans cet exemple, le mot de passe est laissé vide, ce qui signifie qu'aucun mot de passe n'est utilisé. (à changer lors d'une mise en production).
$option C'est un tableau associatif contenant des options de configuration supplémentaires pour la connexion PDO. Ici, une option qui permet de lever les erreurs spécifiques à l'objet PDO (PDOException) et une autre option qui spécifie l'encodage de la base de données (UTF-8 dans notre cas).

La méthode ->prepare($requeteSQL) est utilisée dans le contexte de PDO (appliquée à un objet PDO) pour préparer une requête SQL destinée à être exécutée ultérieurement dans une base de données.

->bindValue(":nom", $nom, PDO::PARAM_STR) Cette ligne de code associe une valeur à un paramètre nommé dans une requête SQL préparée. Voici ce que chaque partie signifie :
":nom" : C'est le nom du paramètre dans la requête SQL préparée. Les paramètres peuvent être nommés, comme dans cet exemple, ou être des marqueurs de position (comme ?). Le préfixe : est souvent utilisé pour les paramètres nommés.
$nom C'est la valeur à associer au paramètre :nom. Dans cet exemple, il semble que $nom contient la valeur à insérer dans la base de données pour le champ nom.
PDO::PARAM_STR C'est le type de données de la valeur associée. Dans ce cas, il s'agit d'une chaîne de caractères. Cela indique à PDO comment traiter la valeur lors de son insertion dans la base de données. PDO::PARAM_STR est utilisé pour les valeurs de type chaîne de caractères. D'autres types de données sont également pris en charge, tels que PDO::PARAM_INT pour les entiers. Autres paramètres PDO
NB: Cette étape est importante pour la sécurité de l'application! (cela évite aussi d'avoir à échapper les caractères spéciaux... )

->execute() est utilisé pour exécuter une requête SQL préparée dans PDO (PHP Data Objects). Si la requête réussit, execute() retourne true. Sinon, il peut retourner false en cas d'échec de l'exécution. (mais ici j'utilise plutôt le PDOStatement pour les exceptions)

La fonction PHP mail($client, $sujet, $message) est utilisée pour envoyer un e-mail à une adresse spécifiée. Voici ce que chaque paramètre signifie :
$client C'est l'adresse e-mail du destinataire.
$sujet C'est le sujet de l'e-mail.
$message C'est le contenu du message à envoyer.
Cette fonction native en PHP envoie un e-mail en utilisant les paramètres fournis. Cependant, il est important de noter que la fonction mail() de PHP dépend de la configuration du serveur pour fonctionner correctement. Il est souvent nécessaire de configurer correctement le serveur pour envoyer des e-mails, par exemple en spécifiant un serveur SMTP dans le fichier de configuration de PHP. De plus, cette fonction peut ne pas fonctionner comme prévu sur tous les serveurs, notamment en raison de problèmes de sécurité ou de restrictions de l'hébergeur.

Le modèle MVC (Modèle-Vue-Contrôleur) est un patron d'architecture utilisé dans le développement logiciel pour organiser une application en trois composants principaux : le modèle (qui gère les données et la logique métier), la vue (qui représente l'interface utilisateur) et le contrôleur (qui gère les entrées de l'utilisateur et gère les interactions entre le Modèle et la Vue). Ce modèle est utilisé par de nombreux Frameworks PHP (Laravel, Symfony, cakePHP etc...)

Dans cette application, le fichier index.php prend ici le rôle de routeur. Suivant la situation, il appelle des fonctions du contrôleur qui lui,(en véritable chef d'orchestre) va faire le lien entre le modèle (qui gère les accès à la BDD) et les différentes vues.

baselayout.php est un template de base qui évite de répéter la structure HTML sur toutes les vues. Les variables $titre et $content de chaque vue sont utilisées dans le template qui est "injecté" dans le code par l'instruction include.
ob_start() et ob_get_clean() sont des fonctions liées au buffer de PHP (mémoire tampon). ob_start() active la mémoire tampon (qui enregistre donc la suite du code) et ob_get_clean() récupère le contenu de la mémoire tampon (sauvegardé ici dans $content)et vide la mémoire tampon (le buffer).

strip_tags(data), stripslashes(data) et htmlentities(data) sont 3 fonctions PHP indispensables pour nettoyer les entrées de l'utilisateur si ceux_ci ne sont pas filtrés par des RegEx (afin de bloquer une partie des codes malveillants).
strip_tags(data) Cette fonction est utilisée pour supprimer toutes les balises HTML et PHP d'une chaîne de caractères. Elle prend en entrée une chaîne de caractères data et retourne une nouvelle chaîne de caractères avec toutes les balises HTML et PHP supprimées.
stripslashes(data) Cette fonction est utilisée pour supprimer les backslashes ( \ ) d'une chaîne de caractères.
htmlentities(data) Cette fonction est utilisée pour convertir les caractères spéciaux en entités HTML dans une chaîne de caractères. Par exemple, elle convertira les symboles tels que < >, &, etc., en entités HTML (<, >, &, etc.), ce qui permet d'afficher correctement du contenu HTML dans un navigateur sans risque d'interprétation incorrecte des balises HTML.
htmlspecialchars(data) (non utilisée ici) est une autre fonction pour convertir les caractères spéciaux en entités HTML dans une chaîne de caractères, tout comme htmlentities(data). Cependant, elle ne convertit pas tous les caractères spéciaux en entités HTML, mais seulement ceux qui sont essentiels pour la structure HTML, et est plutôt limitée à l'encodage Latin1 (ISO-8859-1)

password_hash(data,option) est une fonction PHP utilisée pour générer une fonction de hachage sécurisée.
data représente la donnée à hasher (crypter). option est un paramètre facultatif qui permet de définir le type de cryptage utilisé. Par défaut, si ce paramètre n'est pas spécifié, PHP utilise l'algorithme le plus sécurisé disponible. Cependant, vous pouvez spécifier explicitement l'algorithme en utilisant des constantes comme PASSWORD_DEFAULT, qui est généralement recommandé pour sa sécurité élevée.
La fonction password_verify(password, hash) est utilisée pour confirmer si le mot de passe testé (password) correspond au hachage précédemment généré (hash). Elle renvoie true si la correspondance est vérifiée, sinon false.

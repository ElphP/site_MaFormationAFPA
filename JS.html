React fonctionne selon une hiérarchie de composants enregistrés dans des fichiers .js (ou éventuellement .ts pour du TypeScript). Ceux-ci sont imbriqués les uns dans les autres: le fichier App.js (le fichier principal de l'application) est "rendu" dans le fichier index.js.
Les autres composants (les composants "enfants" de App.js) peuvent être rassemblés dans un dossier "Component" pour une meilleure compréhension de la structure. Par convention les noms des composants prennent une majuscule.

La structure d'un composant React fonctionnel est divisée en plusieurs parties:

  1. une partie import qui comprends tous les imports nécessaires au fonctionnement du composant.
  2. une partie optionnelle dans laquelle il est possible de déclarer des données utilisables en javaScript (ou TS)
  3. Le composant lui-même déclaré par function NomDuComposant { instructions } qui contient lui-même 2 sous-parties (instructions):

    • une partie en JS (ouTS) pour la logique du composant: on peut y trouver du code procédural (boucles, fonctions, etc...), les Hooks (qui définissent la façon dont le composant sera affiché, rendu) exemple de Hook: useState, useEffect etc...,
    • après le mot-clé return une partie en JSX pour la partie affichage . Le JSX est un langage qui présente une similitude marquée avec le HTML. Pour comprendre les codes ci-dessus:

- Passage de props

Le passage de props est un mécanisme fondamental de React pour transmettre des données (variables, fonctions, tableaux, objets...) d'un composant parent à un composant enfant. Les props sont essentiellement des données que vous pouvez fournir à un autre composant pour son utilisation dans une hiérarchie de composants.

Le composant parent transmet un objet props à son composant enfant en utilisant un attribut ajouté à la balise JSX du composant enfant. Cela permet au composant enfant d'accéder aux données fournies par le parent et de les utiliser dans son rendu.

exemple: App.js ligne 17 products={Products}

Pendant longtemps, dans React, l'objet props était récupéré dans le composant enfant de la manière suivante : function MonComposant(props) {...}, puis chaque attribut de props était utilisé dans le corps du composant fonctionnel pour accéder aux différentes valeurs transmises (s'il y en avait plusieurs) de cette manière: props.nom_attribut.

Avec l'introduction d'ES6, la destructuration d'objet est devenue une option pratique pour extraire des valeurs des objets. Ainsi, plutôt que d'utiliser props.nom_attribut, il est maintenant possible de déstructurer l'objet props directement dans les arguments de la fonction du composant enfant, comme ceci : function MonComposant({nom_attribut1, nom_attribut2, etc...}). Chaque nom d'attribut déstructuré correspond aux attributs transmis lors de l'appel du composant enfant dans le composant parent. exemple: ligne 5 de FilterableProductTable => function FiterableProductTable ({products}) {...}
A noter qu'ici le tableau PRODUCTS aurait pu être mis dans ce composant directement car il est juste transmis (au composant enfant "ProductTable" à la ligne 17) au travers de ce composant (FilterableProductTable) sans être utilisé!

- Un premier Hook: useState

Le useState (variable d'état) est un élément de React qui permet à un composant fonctionnel de gérer son propre état interne. Cela permet aux composants fonctionnels d'être dynamiques et interactifs en conservant des données qui peuvent changer au fil du temps. Avec useState, vous pouvez déclarer des variables d'état et les mettre à jour, ce qui déclenche automatiquement un nouveau rendu du composant avec les nouvelles valeurs d'état. C'est un outil essentiel pour créer des interfaces utilisateur réactives dans React.

Pour pouvoir les utiliser (0, 1 ou plusieurs dans un même composant!) il faut:
1- déclarer son utilisation dans le composant avec import exemple: ligne 3 de FilterableProductTable
2- Déclarer la variable d'état en utilisant sa syntaxe particulière (un "getter" et un "setter" déstructurés) exemple: ligne 6 de FilterableProductTable "filterText" est la variable d'état et "setFilterText" est la méthode qui permet de la mettre à jour! La valeur entre ( ) est la valeur initiale de la variable d'état (dans ce cas une chaîne de caractère vide... ).
3- Vous pouvez maintenant utiliser la variable d'état (ne pas oublier les {...} pour l'utilisation dans la partie JSX, comme par exemple dans le composant enfant "SearchBar") ou la mettre à jour avec le "setter" (dans notre cas la fonction "setter" de filterText est passée en props sous le nom "onFilterTextChange" (l.14) au composant enfant "SearchBar" et donc mis à jour dans ce composant! ).

L'utilisation de jQuery peut se faire facilement en utilisant son CDN

jQuery est une bibliothèque JavaScript populaire qui simplifie la manipulation du DOM (Document Object Model) et l'interaction avec JavaScript sur les sites web. Voici quelques intérêts de jQuery :

  1. Sélection d'éléments : jQuery fournit une syntaxe simplifiée pour sélectionner des éléments HTML à l'aide de sélecteurs CSS familiers, ce qui permet de manipuler facilement le contenu de la page.
  2. Manipulation du DOM : jQuery simplifie la modification du contenu HTML, le style des éléments et les interactions avec les événements de la page. Par exemple, vous pouvez ajouter ou supprimer des éléments, modifier du texte ou des attributs, ou attacher des gestionnaires d'événements.
  3. Animations : jQuery fournit des fonctionnalités pour créer des animations sur les éléments HTML, telles que le fondu, le déplacement et le changement de taille.
  4. Requêtes AJAX simplifiées : jQuery facilite l'utilisation d'AJAX pour charger des données à partir du serveur sans recharger la page entière. Il offre des méthodes simples pour effectuer des requêtes AJAX et manipuler les données JSON.
  5. Compatibilité entre navigateurs : jQuery résout de nombreux problèmes de compatibilité entre navigateurs, offrant une API cohérente et prévisible sur différents navigateurs.
  6. Plugins : jQuery dispose d'une vaste communauté de développeurs qui ont créé de nombreux plugins pour étendre ses fonctionnalités. Ces plugins peuvent être utilisés pour ajouter des fonctionnalités telles que des galeries d'images, des menus déroulants, des carrousels, etc.
